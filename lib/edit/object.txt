#
# This file contains information about object kinds.
#
#
# Objects may have descriptions (taken from Oangband).  These descriptions 
# only appear if the player is at least aware of the object kind.  Food, 
# weapons, armors, junk, and tools have complete descriptions, while those 
# of magical devices also use the object flavor (the kind of material) and 
# often insert extra numerical information (indicated by a '~').  Artifacts 
# are described in artifact.txt.
#
# Special thanks to J.R.R Tolkien for fundamental concepts, and to Pat 
# Tracy, a primary source for weapon and armor descriptions.  -LM-
#
# Note that object zero is used for the "stack" picture (currently unused).
#

# Version stamp (required)

V:1.0.2

#
# N: object index: name
# G: ASCII char used to display the object: color symbol : pval
# I: object tval (general kind): object sval (specific type)
# W: object level : activation: 
#      weight (in deci-pounds): value
# M: probability (%) of object appearing in quantity, die roll for number
# A: All depths/abundances of the item (max of three pairs possible)
# C: Base armor class: damage done when wielded or thrown: 
#      Bonus to Skill: Bonus to Deadliness: Bonus to armor class
# F: All item-specific flags (special qualities).  Some depend on pval.
# D: The object descripion, used to provide more information when it is 
#    'I'nspected.
#
# For more information, see ego_item.txt and /lib/info/modify.txt
#

##### Objects 733 to 754 are the "Special Artifacts" #####

# These objects do not specify "full names" because the artifact name is 
# added in "obj-desc.c" based on the artifact index ("i_ptr->artifact_name").
#
# These objects do specify a "base name", which is used when the object is 
# "aware" (always true for light sources).
#
# The light sources (and the One Ring) specify "physical colors", which override 
# the "flavor" colors, if any.  Note that the "One Ring" also has a specific 
# check for "unknown" in which it changes its name to "a plain gold ring".  
# See "obj-desc.c".
#
# Note that the "INSTA_ART" flag is used to prevent these objects from being 
# created without also turning them into artifacts.  This flag must be 
# specified both here and in the artifact template.
#
# The only other data stored in the base object description are the tval and 
# sval.  In particular, level and rarity values are not needed.
#
# The reason for having an object template for each special artifact is to 
# allow each to have a different color and flavor, and also to allow the use 
# of special base names (such as "Necklace").
